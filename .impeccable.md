## Design Context

### Users
- Threads에서 본 글을 빠르게 저장하려는 개인 사용자
- PC에서는 Chrome extension으로 바로 저장하고, 모바일에서는 멘션 기반 scrapbook으로 나중에 모으려는 사용자
- 긴 설명보다 저장 방식과 과금 차이를 빠르게 이해하고 바로 행동하려는 사용자

### Brand Personality
- 명확함
- 절제됨
- 실용적임

이 인터페이스는 군더더기 없이 바로 이해되고, “어떤 상황에서 어떤 저장 방식을 쓰는지”가 즉시 보이는 느낌을 목표로 한다.

### Aesthetic Direction
- 밝은 배경의 미니멀 제품 랜딩
- 과장된 장식, 복잡한 카드 중첩, 과한 색 포인트는 피함
- 정보 구조는 `PC용 확장`, `모바일용 멘션`, `Free vs Pro` 세 축으로 정리
- 스크린샷은 설명을 대신하는 증거로 사용

### Design Principles
1. 첫 화면에서 제품 구조를 한 번에 이해시킨다.
2. 텍스트보다 구분된 흐름과 비교표로 설명한다.
3. 장식보다 여백, 정렬, 타이포그래피로 완성도를 만든다.
4. CTA는 섹션마다 하나만 강하게 두고 나머지는 보조로 둔다.
5. 모바일과 데스크톱 사용 맥락을 섞지 않고 분리해서 보여준다.
