## Design Context

### Users
- Threads에서 본 글을 빠르게 저장하려는 개인 사용자
- PC에서는 Chrome extension으로 바로 저장하고, 모바일에서는 멘션 기반 scrapbook으로 나중에 모으려는 사용자
- 긴 설명보다 저장 방식과 과금 차이를 빠르게 이해하고 바로 행동하려는 사용자

### Brand Personality
- 명확함
- 절제됨
- 실용적임

이 인터페이스는 군더더기 없이 바로 이해되되, 실제 사용 장면이 먼저 보이면서 "아, 이 서비스가 정확히 뭘 하는지"를 5초 안에 이해시키는 느낌을 목표로 한다.

### Aesthetic Direction
- 밝은 배경, 진한 타이포, 얇은 구분선 중심의 문서형 랜딩
- 과장된 장식이나 장면 과시는 줄이고, 제목·요약·목록만으로 핵심이 바로 읽히게 구성
- 정보 구조는 `Hero -> PC 저장 -> 모바일 저장 -> 저장 이후 활용 -> Free vs Plus -> Trust` 순서로 정리
- PC 확장과 모바일 멘션은 하나의 제품 안에 있는 두 저장 경로로 보이되, 행동 선택은 즉시 가능해야 함
- 화면을 잘게 나누는 박스형 구성을 피하고, 여백과 구분선으로 섹션을 나눔

### Design Principles
1. 첫 화면에서 서비스 정체성과 저장 경로를 동시에 이해시킨다.
2. 텍스트 설명보다 실제 사용 장면과 흐름으로 설득한다.
3. PC 확장과 모바일 멘션은 반드시 시각적으로 분리해 보여준다.
4. 저장 이후의 가치와 Plus 업셀은 한눈에 비교 가능해야 한다.
5. 설치, 결제, 데이터 처리에 대한 불안을 평문과 신뢰 신호로 바로 해소한다.
