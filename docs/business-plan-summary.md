# Threads 저장 도구 사업계획서 요약

## 1. 사업 정의

- 제품명(현 단계): `Threads to Obsidian`
- 제품 정의: Threads 개별 글과 작성자 자기 답글 체인을 PC 환경에서 구조화해 개인 지식 도구로 저장하는 Chrome 확장
- 현재 핵심 가치: `소셜 피드를 소비로 끝내지 않고 개인 지식 자산으로 전환`
- 현재 핵심 저장 대상: `Obsidian`
- 다음 확장 대상: `Notion`
- 모바일 전략: 별도 2단계. 본 계획서는 `PC 환경` 기준으로 한정

## 2. 문제 정의

- Threads에는 짧지만 재활용 가치가 높은 아이디어, 관찰, 업계 정보, 자기 메모형 포스트가 많다.
- 하지만 Threads 자체는 검색, 장기 보관, 재편집, 개인 지식베이스 통합이 약하다.
- 사용자는 현재 수동 복사, 스크린샷, 링크 저장, 북마크, 메모앱 수기 정리에 의존한다.
- 특히 자기 답글이 이어지는 포스트는 본문과 후속 맥락이 분리되어 저장 품질이 낮다.
- 결과적으로 `발견은 쉽고 축적은 어렵다`는 문제가 존재한다.

## 3. 해결책

- PC 브라우저에서 Threads 개별 포스트를 열고 확장 아이콘을 누르면 저장이 실행된다.
- 본문, 작성자, 게시 시각, 원문 URL, 이미지, 작성자 자기 답글을 하나의 저장 단위로 묶는다.
- 사용자의 지식 도구 구조에 맞게 파일명, 저장 경로, 메타데이터를 자동 정리한다.
- 직접 저장이 안 되는 환경에서도 ZIP 다운로드로 폴백해 저장 실패를 줄인다.
- 장기적으로는 Obsidian뿐 아니라 Notion 같은 구조화된 문서 도구까지 같은 흐름으로 지원한다.

## 4. 제품 범위

### 4.1 현재 제공 가치

- Threads 개별 포스트 저장
- 작성자 자기 답글 체인 통합 저장
- 이미지/미디어 포함 저장
- ZIP 다운로드 폴백
- 중복 저장 방지
- 한국어/영어 UI
- Pro 라이선스 활성화 및 결제/키 발급 운영
- BYO API Key 기반 AI 요약/태깅

### 4.2 제품의 본질

- 이 제품은 단순 다운로더가 아니다.
- `소셜 글 수집 -> 구조화 -> 개인 문서도구 적재`라는 워크플로우 제품이다.
- 차별점은 범용 웹 클리퍼가 아니라 `Threads 특화 추출 품질`과 `작성자 후속 답글 보존`에 있다.

## 5. 시장 및 타겟

### 5.1 1차 타겟

- Obsidian 사용자
- PKM, Second Brain, 리서치 워크플로우 사용자
- Threads에서 아이디어와 업계 동향을 저장하는 사용자
- 개발자, 마케터, 작가, 창업자, 크리에이터

### 5.2 2차 타겟

- Notion을 문서/아카이브 용도로 쓰는 사용자
- 개인용 지식 저장보다 `정리된 문서 페이지`를 선호하는 사용자
- 팀 단위가 아니라 우선 개인 저장 수요가 강한 사용자

### 5.3 제외 범위

- 모바일 우선 사용자
- Threads 전체 피드 백업 수요
- 엔터프라이즈 협업/관리 기능 수요
- 소셜 마케팅 자동 게시 도구 수요

## 6. 왜 PC 우선인가

- 현재 핵심 저장 행위는 브라우저 확장과 폴더 권한, ZIP 다운로드, 로컬 파일 구조에 의존한다.
- Obsidian 직접 저장은 PC에서만 제품 가치가 완전하게 드러난다.
- PC 사용자는 저장 후 즉시 편집, 태깅, 재배치, 링크 연결을 수행할 수 있다.
- 개발 비용 대비 제품 완성도와 전환율이 PC에서 더 유리하다.
- 따라서 PMF 이전에 모바일까지 동시에 확장하면 품질과 일정이 모두 무너질 가능성이 높다.

## 7. 포지셔닝

- 1차 포지셔닝: `Threads를 Obsidian에 가장 잘 저장하는 도구`
- 2차 포지셔닝: `Threads를 PC 지식 도구로 보내는 저장 도구`
- 장기 포지셔닝: `Threads knowledge capture layer`

경쟁 대안은 다음과 같다.

- 수동 복사/붙여넣기
- 스크린샷 저장
- 북마크/읽기 목록
- 범용 웹 클리퍼
- Readwise 같은 허브형 서비스

우리의 포지셔닝 우위는 다음이다.

- Threads 문법에 맞는 추출
- 작성자 답글 체인 보존
- 저장 결과가 바로 재사용 가능한 구조
- 사용자의 기존 지식 도구를 대체하지 않고 연결

## 8. 상품 전략

### 8.1 Free

- Threads 포스트 저장
- 작성자 자기 답글 통합
- 이미지 저장
- ZIP 다운로드
- Obsidian 직접 저장
- 기본 메타데이터
- 기본 중복 처리

### 8.2 Pro

- 파일명 규칙
- 저장 경로 규칙
- 다중 저장 위치 확장 기반
- 고급 정리 옵션
- 향후 Notion 대상 저장
- 향후 추가 PC 저장 대상 지원

### 8.3 AI Add-on 성격

- AI는 독립 상품이 아니라 저장 후 정리 강화 기능이다.
- 사용자가 API Key를 직접 넣는 BYO Key 모델을 유지해 원가 부담을 낮춘다.
- AI는 후킹 포인트이지만 핵심 가치는 아니다.

## 9. Notion 확장 전략

### 9.1 판단

- Notion은 Obsidian 다음으로 확장 가치가 큰 저장 대상이다.
- 이유는 사용층이 크고, `문서 페이지로 저장`하는 가치 제안이 명확하기 때문이다.
- 단, Obsidian과 달리 로컬 파일 저장이 아니라 API 기반 문서 생성이므로 제품 모델이 달라진다.

### 9.2 1차 Notion 범위

- PC 환경에서만 지원
- 사용자가 Notion internal integration token과 parent page를 설정
- Threads 포스트를 새 Notion 페이지로 저장
- YAML frontmatter 없이 Notion 전용 문서 형식으로 저장
- 작성자, 원문 URL, 본문, 답글, 이미지 링크를 포함

### 9.3 2차 Notion 범위

- database/data source 저장
- page properties 매핑
- 재저장/업데이트 흐름
- 파일 업로드
- 장기적으로 OAuth/public integration 지원

### 9.4 사업적 의미

- Obsidian 한정 제품에서 `저장 대상 확장형 제품`으로 포지션을 넓힌다.
- 단일 커뮤니티 의존도를 줄인다.
- Pro 업그레이드 근거가 더 강해진다.

## 10. 모바일 전략

- 모바일은 후속 단계다.
- 모바일에서 같은 가치를 내려면 브라우저 확장이 아니라 `공유 시트 기반 저장` 모델이 필요하다.
- 저장 대상, 인증, 이미지 처리, 앱 생명주기, Obsidian/Notion 연결 방식이 전부 달라진다.
- 따라서 본 사업계획서는 모바일을 매출 예측과 개발 일정의 핵심 가정에 넣지 않는다.
- 모바일 착수 조건은 다음과 같다.
  - PC 버전 저장 성공률과 전환율이 안정화
  - Obsidian + Notion 중 최소 2개 대상 지원
  - 고객 문의에서 모바일 수요가 반복적으로 확인

## 16. 핵심 리스크

- Threads DOM 변경으로 추출 품질이 깨질 수 있다.
- Chrome Web Store 정책과 실제 동작 문구가 어긋나면 심사 리스크가 생긴다.
- Obsidian 단일 시장만 보면 시장 크기 한계가 있다.
- Notion 지원은 가능하지만 인증, API 제약, 문서 모델 차이로 구현 복잡도가 높다.
- 모바일을 너무 빨리 확장하면 제품 품질과 개발 집중도가 무너질 수 있다.

## 17. 대응 전략

- 파서 구조를 테스트 기반으로 유지하고 fallback 로직을 강화
- 저장 실패 로그와 사용자 제보를 빠르게 패치 루프로 반영
- 스토어 문구, 개인정보 처리방침, 권한 사용 설명을 항상 실제 동작과 일치
- 저장 타깃 어댑터 구조로 제품을 재정리해 Obsidian 이후 확장을 쉽게 만듦
- 모바일은 후순위로 두고 PC에서 PMF를 먼저 검증

## 18. 최종 판단

- 이 사업의 핵심은 `AI 요약`이 아니라 `신뢰할 수 있는 저장`이다.
- 단기적으로는 `PC용 Obsidian 저장 도구`로 시장을 확보하는 것이 맞다.
- 중기적으로는 `Notion 지원`을 붙여 제품 범위를 넓히는 것이 가장 타당하다.
- 모바일은 별도 제품 단계로 보고, PC 버전의 성공 이후에 착수한다.
- 따라서 현재 사업의 우선순위는 다음 순서다.

1. PC Obsidian 저장 품질 완성
2. PC Notion 저장 도입
3. Pro 가치와 가격 방어 강화
4. 모바일 검증 착수
